## Çözülen problemler (bu aralıkta kapatıldı)

### P-1 · Loot pickup non-transactional idi (P02)

- Commit: `aa585e1`
- Problem: drop claim + cargo mutate + XP grant ayrı adımlardı; arada hata/retry duplicate XP
  veya tutarsız cargo/claim bırakabiliyordu.
- Çözüm: `loot/service.go pickupDropWithTransaction` → `pendingClaims` guard altında claim alınır,
  ardından `WithLootPickupTransaction` içinde `SaveLootDropClaim + CommitInventoryAddItem +
InsertOutboxRow` atomik commit edilir. Hata halinde `RestoreMutationState` + `clearPendingClaim`
  ile in-memory cargo geri alınır. XP commit sonrası canonical `loot_pickup:<drop_id>`
  idempotency key'li outbox row'dan replay edilir → çift XP yok.
- Kanıt: `loot/pickup_transaction_test.go::TestPickupDropTransactionOutboxFailureRollsBackClaimAndCargo`,
  migration `0017_loot_drop_claims_schema.sql`.

### P-2 · Premium claim/provider durable + replay-safe değildi (P02)

- Commit: `aa585e1`
- Problem: premium entitlement state in-memory'di; provider event replay ve claim retry çift
  entitlement / çift wallet credit üretebilirdi.
- Çözüm: `premium/service.go` `createEntitlementWithTransactionLocked` /
  `claimEntitlementWithTransactionLocked` → `FOR UPDATE` row lock + idempotency claim/complete +
  wallet commit + outbox tek transaction. Provider tekilliği
  `UNIQUE (provider_source, provider_reference)` + idempotency ile sağlanır.
- Kanıt: `premium/service_test.go` (provider replay, currency-pack commit, injected-outbox
  rollback), `contentdb/premium_postgres_smoke_test.go`, migration
  `0016_premium_entitlement_state_schema.sql`.
